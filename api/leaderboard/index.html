Leaderboard API

Public, read-only JSON endpoints for game leaderboard scores. No authentication required. Updated daily.

Public · Read-Only · No Auth

Overview

PropertyDetails
Base URLhttps://vaultsparkstudios.com/api/leaderboard/v1/
FormatJSON
AuthNone required (public endpoints)
CORSAccessible cross-origin (static files on GitHub Pages)
Rate LimitsGitHub Pages defaults (~100 req/min per IP). Data refreshes daily.
VersioningURL-based (/v1/). Breaking changes will use a new version path.

Endpoints

All Games — Top 100

GET /api/leaderboard/v1/all.json

Top 100 scores across all VaultSpark games, sorted by score descending.

Per-Game Leaderboards — Top 50

GET /api/leaderboard/v1/call-of-doodie.json

Top 50 scores for Call of Doodie.

GET /api/leaderboard/v1/gridiron-gm.json

Top 50 scores for Gridiron GM.

GET /api/leaderboard/v1/vaultspark-football-gm.json

Top 50 scores for VaultSpark Football GM.

API Metadata

GET /api/leaderboard/v1/meta.json

Available endpoints, game list, and last-updated timestamp.

Response Format

Each game endpoint returns an object with the following shape: 

{
  "game": "call-of-doodie",
  "game_name": "Call of Doodie",
  "updated_at": "2026-03-27T06:15:00.000Z",
  "entries": [
    {
      "rank": 1,
      "username": "player1",
      "score": 5000,
      "rank_title": "Vault Keeper"
    }
  ]
}

Entry Fields

FieldTypeDescription
ranknumberPosition on the leaderboard (1-indexed)
usernamestringVault Member display name
scorenumberHighest recorded score
rank_titlestringVault rank tier (e.g. "Vault Keeper", "Rift Scout")

The all.json endpoint adds game and game_name fields to each entry so you can distinguish which game a score belongs to.

Embed Widget

Drop this snippet into any webpage to show a live leaderboard. No API key needed. 

<div id="vaultspark-leaderboard"
     data-game="call-of-doodie"
     data-limit="10"></div>
<script src="https://vaultsparkstudios.com/api/leaderboard/v1/widget.js"></script>

Widget Options

AttributeDefaultDescription
data-game"all"Game slug: all, call-of-doodie, gridiron-gm, vaultspark-football-gm
data-limit25Number of entries to show (max depends on endpoint: 100 for all, 50 per game)

Usage Examples

JavaScript (fetch)

fetch('https://vaultsparkstudios.com/api/leaderboard/v1/call-of-doodie.json')
  .then(r => r.json())
  .then(data => {
    data.entries.forEach(e =>
      console.log(`#${e.rank} ${e.username}: ${e.score}`)
    );
  });

Python

import requests

data = requests.get(
    "https://vaultsparkstudios.com/api/leaderboard/v1/all.json"
).json()

for entry in data["entries"]:
    print(f"#{entry['rank']} {entry['username']}: {entry['score']}")

cURL

curl -s https://vaultsparkstudios.com/api/leaderboard/v1/meta.json | jq .

Notes

  • Data is regenerated daily via GitHub Actions. Scores may be up to 24 hours behind real-time.
  • For real-time scores, use the VaultScore SDK (requires Vault Membership sign-in).
  • The API is read-only. To submit scores, players must be signed-in Vault Members playing through official game pages.
  • Empty leaderboards will return an entries array with zero elements.
  • This API uses semantic versioning in the URL path. The current version is v1.
