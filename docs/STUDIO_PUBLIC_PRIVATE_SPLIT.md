# VaultSpark Public and Private Split

This public repo should be treated as the public-safe studio layer, not the full
internal studio brain.

## Public repo role

Repo: `VaultSparkStudios.github.io`

Keep here:

- public website content
- public-facing brand and organization information
- public deployment references
- sanitized public-safe templates
- lightweight high-level methodology docs

## Private repo role

Repo: `vaultspark-studio-ops`

Keep there:

- internal studio-system source of truth
- IP-sensitive templates
- rights and provenance systems
- creative direction history
- innovation systems
- canon and franchise systems
- approval and governance materials
- internal prompts and unpublished strategy

## Rule

New sensitive studio docs should be created in the private ops repo or in a
private project repo, not here.

## Transition note

This public repo may still contain legacy copies of some internal-facing docs or
templates. Do not expand those here. Treat the private repo as the forward
source of truth for sensitive materials.
