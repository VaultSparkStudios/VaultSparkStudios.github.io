# VaultSpark Document Authority

This document defines which files are authoritative when records disagree.

## Authority classes

### Class A: Identity and canon

Examples:

- `SOUL.md`
- `CANON.md`
- `BRAND_SYSTEM.md`
- `RIGHTS_PROVENANCE.md`
- `CREATIVE_DIRECTION_RECORD.md`

### Class B: Current operating truth

Examples:

- `CURRENT_STATE.md`
- `TASK_BOARD.md`
- `LATEST_HANDOFF.md`

### Class C: Decision history

Examples:

- `DECISIONS.md`
- `RETCON_LOG.md`
- `LEARNING_LOOP.md`
- `WORK_LOG.md`

### Class D: Exploratory or temporary material

Examples:

- brainstorming notes
- scratch prompts
- chat transcripts

## Default precedence

When records conflict, use this order:

1. Explicit user instruction
2. Current production code or shipped artifact
3. Class A identity / canon files
4. Class B current operating files
5. Class C historical files
6. Class D exploratory notes

## Escalate instead of guessing

Escalate before making unilateral changes that affect:

- project identity
- canon
- rights or provenance
- public promises
- launch commitments
- security or data handling
