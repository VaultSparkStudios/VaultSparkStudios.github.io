# VaultSpark New Project Bootstrap

Use this process for every new serious project.

## Before code or production work starts

1. Create the repo with the canonical public slug
2. Add `AGENTS.md`
3. Create the baseline folders:
   - `context/`
   - `docs/`
   - `plans/`
   - `specs/`
   - `logs/`
   - `handoffs/`
   - `prompts/`
4. Create the baseline project files from templates
5. Add only the medium-specific templates that project needs

## Fill these first

- `context/PROJECT_BRIEF.md`
- `context/BRAIN.md`
- `context/SOUL.md`
- `context/CURRENT_STATE.md`
- `context/TASK_BOARD.md`
- `context/LATEST_HANDOFF.md`
- `prompts/start.md`
- `prompts/closeout.md`
- `AGENTS.md`

## Then add higher-leverage records

- `context/OPEN_QUESTIONS.md`
- `context/ASSUMPTIONS_REGISTER.md`
- `context/RISK_REGISTER.md`
- `docs/CREATIVE_DIRECTION_RECORD.md`
- `prompts/PROMPT_LIBRARY.md`
- `logs/WORK_LOG.md`
