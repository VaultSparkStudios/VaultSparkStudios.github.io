VaultSpark Studios  ·  Security

Security Policy

Last updated: April 3, 2026

The security of our users, their data, and our systems is a top priority at VaultSpark Studios. This page explains how to report security vulnerabilities and describes the security measures we have in place.

Report a vulnerability:

Email founder@vaultsparkstudios.com with subject Security Report

Responsible Disclosure

We believe in working with security researchers to keep our users safe. If you have discovered a security vulnerability in any VaultSpark Studios service, we appreciate your help in disclosing it to us responsibly.

How to Report

Send an email to founder@vaultsparkstudios.com with the subject line "Security Report" and include:

  • A description of the vulnerability and its potential impact
  • Steps to reproduce the issue
  • The affected URL, endpoint, or repository
  • Any proof-of-concept code or screenshots
  • Your preferred contact method for follow-up

What We Promise

  • We will acknowledge your report within 3 business days
  • We will provide a status update within 10 business days
  • We will not take legal action against researchers who follow this policy
  • We will credit you (if desired) when the vulnerability is resolved

What We Ask

  • Give us reasonable time to fix the issue before public disclosure (90 days)
  • Do not access, modify, or delete data belonging to other users
  • Do not perform denial-of-service attacks or degrade service quality
  • Do not use automated scanning tools against production without prior coordination
  • Act in good faith — avoid privacy violations and data destruction

Scope

The following are in scope for responsible disclosure:

  • vaultsparkstudios.com and all subdomains
  • VaultSpark Studios GitHub repositories (public repos)
  • VaultSpark APIs and backend services
  • Vault Member authentication and account systems

Out of Scope

  • Third-party services we use (Supabase, Google Analytics, GitHub) — report to them directly
  • Social engineering or phishing attacks against staff
  • Denial-of-service vulnerabilities without practical exploit
  • Issues already reported or publicly known

Our Security Practices

Infrastructure

  • HTTPS everywhere — all traffic is encrypted via TLS
  • Content Security Policy — strict CSP headers on every page to prevent XSS
  • Security headers — X-Content-Type-Options, X-Frame-Options, Referrer-Policy, Permissions-Policy
  • Static hosting — the main site is static HTML on GitHub Pages with no server-side execution surface

Authentication

  • Vault Member authentication is handled by Supabase Auth with bcrypt password hashing
  • Session tokens are stored securely and rotated on refresh
  • Rate limiting is applied to authentication endpoints

Data Protection

  • No advertising trackers or third-party marketing pixels
  • Minimal data collection — we only store what is necessary
  • Regular security reviews of dependencies and configurations
  • Error tracking via Sentry with PII scrubbing enabled

Code Security

  • All code changes go through review before deployment
  • GitHub Dependabot monitors for vulnerable dependencies
  • CI/CD pipelines include automated security checks

For non-security inquiries, use our contact page. For security reports, email founder@vaultsparkstudios.com with subject "Security Report".
